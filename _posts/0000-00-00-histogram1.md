## Set up CMake

> Add a `CMakeLists.txt` file:
{:.block-task}

```cmake
cmake_minimum_required(VERSION 3.14)
project(Exercises)

set(CMAKE_CXX_STANDARD 26)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_executable(hist1 hist1.cpp)
```

> Optional: Improve [LSP][3] support
>
> If you use `clangd` as an [LSP][3], add a `.clangd` file that points to GCC 16's libstdc++:
> ```
> CompileFlags:
>  Add:
>    - --gcc-toolchain=/home/mkretz/.local/gcc-16
> ```
> And add `set(CMAKE_EXPORT_COMPILE_COMMANDS ON)` to `CMakeLists.txt`.
>
> That way CMake will generate `compile_commands.json` file, and `clangd` knows exactly how to compile your code as you go.
>
> (I use the `ale` plugin for `vim`.)
{:.block-tip}

> Add everything to the git repo:
{:.block-task}
```sh
git add CMakeLists.txt .clangd
git commit
```

> Extend `CMakeLists.txt` with a build-and-run target
>
> 1. Define a `function(add_runnable name)` which calls `add_executable`
> 2. Change `add_executable(hist1 hist1.cpp)` to `add_runnable(hist1)`.
> 3. Use `add_custom_target` to add a `run_${name}` target.
> 4. `cmake --build build -- run_hist1` builds and executes in one step.
> 5. Alternative: `cd build; make run_hist1`

## Performance

> Let's consider performance.
>
> 1. Build with `VERBOSE=1` (or `-v` to `cmake --build`) to see the call to `g++`.
>    Inspect the compiler flags.
> 2. `ls -l build/hist1`: Note down the binary size.
> 3. Run `hyperfine build/hist1` (if the time is below 300ms, increase `N`)
> 4. Run `perf stat build/hist1` (prepend `taskset 1` to inhibit cpu-migrations)
{:.block-warning}

> Configure in "Release" mode
>
> 1. CMake defaults to an empty `CMAKE_BUILD_TYPE`
> 2. use `ccmake` in the `build` directory to set it to `Release`
> 3. Redo `hyperfine` and `perf stat`
{:.block-tip}

> CMake defaults
>
> I disagree with the defaults
> - `Release` or `RelWithDebInfo` should be default
> - `CMAKE_CXX_FLAGS_RELEASE` defaults to `-O3 -DNDEBUG`; should be `-O2 -DNDEBUG`
> - `CMAKE_CXX_FLAGS_RELWITHDEBINFO` defaults to `-O2 -g -DNDEBUG`; should be `-Og -g -DNDEBUG`
>
> A very interesting flag to add nowadays is [`-fhardened`][4]
{:.block-warning}

> Use the right tools
>
> - Did you use a `std::map`, as in the cppreference example? Consider a `std::vector<int>`.
> - Try with `std::list` and see what `perf stat` tells you.
{:.block-tip}
