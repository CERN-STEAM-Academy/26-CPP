In the `simd-benchmarks` directory:

> CMake
>
> Edit `CMakeLists.txt` and add `add_benchmark(peakflop)` at the bottom.
{:.block-task}

> Code
>
> Create a new file `peakflop.cpp` and add the following boilerplate:
{:.block-task}
```c++
#include <benchmark/benchmark.h>

void peak(benchmark::State &state)
{
  float x = 1;
  for (auto _ : state) {
    x = x * 3 + 1;
  }

  // compute FLOP/s and FLOP/cycle
  constexpr double flop_per_iteration = 2;
  state.counters["FLOP"] = {flop_per_iteration,
                            benchmark::Counter::kIsIterationInvariantRate};
  if (state.counters.contains("CYCLES")) {
    state.counters["FLOP/cycle"] = {flop_per_iteration / state.counters["CYCLES"],
                                    benchmark::Counter::kIsIterationInvariant};
  }
}

// Register the function as a benchmark
BENCHMARK(peak);

// Run the benchmark
BENCHMARK_MAIN();
```

> Run the benchmark
>
> Use `make run_peakflop` to compile and execute the benchmark (run `make` on 
the first time to create the new target)
{:.block-task}

> Are the numbers correct?
>
> Anything wrong with the benchmark?
{:.block-warning}

> Inspect the binary
>
> Inspect the binary with
> ```sh
> vir_inspect.sh peakflop peak
> ```
{:.block-task}

> Inspect with Compiler Explorer
>
> Inspect the example we benchmarked using Compiler Explorer. Remove FLOP/s 
computation. [Short link](https://compiler-explorer.com/z/e7fafqdbM)
{:.block-task}

> TIP
>
> Use e.g. `std::vector<int>` in place of `benchmark::State`.
{:.block-tip}

> Fix the benchmark
>
> Modify the benchmark to produce believable results.
{:.block-task}

> TIP
>
> [:green_book: `benchmark::DoNotOptimize(x)`][7]
{:.block-tip}

> Inline assembly barrier
>
> Or invoke inline assembly yourself:
> ```c++
> asm volatile("" : "+x"(x));
> ```
> It is different from `benchmark::DoNotOptimize`. Is it better? more correct? 
Discuss.
{:.block-task}

> Local Compiler Explorer
>
> Of course you can achieve a very similar result locally, using e.g. the
> following command. Compiler Explorer has the added feature of better
> annotation of the assembler output and easy testing of different compilers and
> compiler flags.
>
> ```sh
> CXXFLAGS=-std=gnu++2b -O2 -DNDEBUG
> watch "ccache g++ $CXXFLAGS -c -S -o - -masm=intel myfile.cpp|grep -vE '^\s+\.'|c++filt"
> ```
>
> Drop `ccache` if you don't have it available. But since `watch` recompiles
> every 2s, caching recompiles of unchanged code is not such a bad idea.
{:.block-tip}
